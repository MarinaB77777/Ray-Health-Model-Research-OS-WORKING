# Health Model Data Contract

## Purpose

This document defines the ownership, boundaries, lifecycle, and separation rules for Health Model Data within the psychophysical model ecosystem.

The purpose of this contract is:

- define Health Model Data ownership;
- define Health Model Data boundaries;
- preserve source-data provenance;
- preserve Health Model independence;
- prevent hidden authority accumulation;
- prevent domain collapse;
- support explainable analysis.

This document governs:

- Health Model source data;
- Health Model calculated data;
- Health Model ownership boundaries;
- Health Model consumption boundaries;
- Health Model analysis boundaries;
- Health Model export boundaries.

This document does NOT:

- define Health Model formulas;
- define Runtime behavior;
- define Research Layer behavior;
- define Governance permissions;
- define Session Lifecycle rules;
- define Retention policy;
- define Consent policy.

---

# Health Model Domain

The Health Model domain is currently implemented in:

model_engine/

The Health Model domain owns:

- Health Model source data;
- Health Model calculated outputs;
- Health Model state outputs;
- Health Model uncertainty outputs;
- Health Model forecast-governance outputs.

The Health Model remains an independent domain.

---

# Core Principle

Health Model Data belongs to the Health Model domain.

Other layers may reference Health Model Data.

Other layers do not own Health Model Data.

Consumption
≠
ownership transfer.

Reference
≠
ownership transfer.

# Health Model Data

Health Model Data includes:

## Source Data

Data acquired for Health Model evaluation.

Examples:

- questionnaire answers;
- adaptive dialogue answers;
- acquisition responses;
- calibration inputs used by the model;
- sensor snapshots used by the model.

Source Data remains source information.

Source Data
≠
calculated result.

# Readiness Boundary

readiness/

is:

- readiness layer;
- analysis layer;
- uncertainty layer;
- consistency layer.

readiness/

is NOT:

- Health Model Core;
- Runtime;
- Research Export;
- Governance authority.

Readiness may analyze Health Model outputs.

Readiness does not own Health Model outputs.

Readiness does not rewrite Health Model outputs.

Readiness does not rewrite Health Model source data.

---

# Research Boundary

Research layers may:

- analyze Health Model outputs;
- compare Health Model outputs;
- correlate Health Model outputs with other domains;
- generate findings;
- generate hypotheses.

Research findings
≠
Health Model state.

Research findings
≠
Health Model truth.

Research outputs
≠
Health Model ownership.

Research outputs must remain separate from Health Model storage.

---

# Runtime Boundary

Runtime may:

- request data;
- route acquisition;
- route communication;
- coordinate execution.

Runtime decisions
≠
Health Model outputs.

Runtime state
≠
Health Model state.

Runtime must not silently rewrite Health Model records.

---

# Governance Boundary

Governance may:

- evaluate permissions;
- evaluate visibility;
- evaluate export boundaries;
- evaluate privacy boundaries.

Governance verdict
≠
Health Model output.

Governance restrictions
≠
Health Model calculation.

Governance does not own Health Model Data.

---

# Feedback Boundary

Research outputs may reference source domains.

Research outputs must not overwrite source domains.

Research outputs must not silently modify source-domain records.

Finding generated
≠
source data updated.

Hypothesis generated
≠
Health Model updated.

Correlation detected
≠
Health Model rewritten.

---

# Acquisition Mode Independence

The system may support multiple acquisition modes.

Examples:

- Questionnaire Mode;
- Adaptive Dialogue Mode.

Acquisition mode affects collection.

Acquisition mode does not affect ownership.

Acquisition mode does not affect provenance.

Acquisition mode does not create a new Health Model domain.

---

# Questionnaire Mode

Questionnaire Mode is an acquisition route.

Questionnaire Mode may produce Health Model source data.

Questionnaire Mode does not determine domain ownership.

Questionnaire Mode
≠
Health Model domain.

---

# Adaptive Dialogue Mode

Adaptive Dialogue Mode is an acquisition route.

Adaptive Dialogue Mode produces Health Model source data.

Adaptive Dialogue Mode
≠
Health Model domain.

---

# Storage Rules

Health Model source records may be corrected only with provenance preservation.

Correction
≠
replacement of historical origin.

Health Model calculated outputs may be recalculated.

Corrections must preserve provenance.

Recalculation must preserve traceability.

Loss of provenance is considered architectural degradation.

---

# Export Boundary

Health Model Data
≠
Research Export

Health Model Data
≠
Participant Export

Exports may contain filtered representations of Health Model outputs.

Exports do not automatically contain all internal Health Model structures.

Internal calculations may remain internal.
