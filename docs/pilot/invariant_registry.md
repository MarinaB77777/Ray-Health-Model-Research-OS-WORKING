# Invariant Registry

## Purpose

This document defines canonical invariants used across Runtime, Research, Governance, Export, Sensor, and Schema contracts.

The purpose of this registry is:
- semantic consistency;
- invariant stability;
- anti-drift governance;
- epistemic discipline;
- reduced wording divergence;
- explainable architectural boundaries.

This registry exists to reduce:
- duplicated semantic definitions;
- contradictory invariants;
- hidden authority drift;
- interpretation ambiguity;
- governance entropy.

This document does NOT:
- replace domain contracts;
- override domain-specific semantics;
- act as Runtime authority;
- act as Governance authority;
- centralize all semantic interpretation.

---

# Core Principle

Canonical invariants should remain:
- stable;
- explicit;
- reviewable;
- version-aware;
- explainable;
- semantically bounded.

Invariants are:
- architectural semantic anchors;
- governance stabilization tools;
- epistemic boundary references.

Invariants are NOT:
- unrestricted truth claims;
- hidden authority mechanisms;
- semantic shortcuts for removing uncertainty;
- universal semantic override rules.

Invariant Registry
≠
semantic single-source-of-truth.

Domain contracts remain primary semantic authority.
