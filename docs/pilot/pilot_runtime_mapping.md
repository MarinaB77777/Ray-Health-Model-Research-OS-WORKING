# Pilot Runtime Mapping

## Purpose

This document defines the bounded operational topology for the university pilot version of the psychophysical readiness system.

The purpose of this mapping is:
- Runtime separation clarity;
- explainable pilot orchestration;
- bounded operational flow;
- anti-hidden-authority architecture;
- pilot-safe Runtime coordination;
- uncertainty-aware loops;
- blocked/unresolved state visibility.

This document defines:
- operational layer responsibilities;
- pilot Runtime flow;
- bounded orchestration topology;
- clarification/acquisition loops;
- unresolved and blocked paths;
- operational truth boundaries.

This document does NOT:
- define autonomous Runtime;
- define unrestricted orchestration;
- define Inner Core access;
- define hidden adaptive authority;
- define unrestricted agent behavior;
- define deterministic truth synthesis.
